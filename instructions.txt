Esse objeto é o retorno de uma execução de um agente/conversa usando LangChain (ou LangGraph) com suporte a ferramentas (tools).
Ele é um dicionário com a seguinte estrutura principal:

    'messages': [lista de mensagens]

Dentro de `'messages'` temos uma lista que representa o histórico completo da conversa, passo a passo:

| Ordem | Tipo de Mensagem     | O que significa |
|------|----------------------|-----------------|
| 0    | `HumanMessage`       | Mensagem do usuário ("que dia é hoje?") |
| 1    | `AIMessage`          | Resposta inicial da IA → ela decidiu chamar uma tool (`get_date`) |
| 2    | `ToolMessage`        | Resultado da ferramenta executada ("2026-06-07") |
| 3    | `AIMessage`          | Resposta final da IA depois de ver o resultado da tool |

Esse padrão é muito comum em agentes com ferramentas (ReAct, tool calling, etc.). A IA pensa, chama ferramenta, recebe o resultado e depois gera a resposta final.

---

### Por que usar `print(response['messages'][-1].content)`?

Porque você geralmente só quer a resposta final que deve ser mostrada para o usuário.

- `response['messages']` → lista completa do histórico
- `[-1]` → pega o último item da lista (o mais recente)
- `.content` → extrai apenas o texto da mensagem

No exemplo que você mostrou, `response['messages'][-1].content` retorna:

```python
"Hoje é sábado, dia 7 de junho de 2026! 😊"
```

### Alternativas e boas práticas

```python
# Forma mais comum e recomendada
final_message = response['messages'][-1]
print(final_message.content)

# Ou ainda melhor (mais robusto):
if hasattr(response, 'messages'):
    print(response['messages'][-1].content)
```

Você também pode querer mais informações:

```python
last_msg = response['messages'][-1]

print("Conteúdo:", last_msg.content)
print("Tipo:", type(last_msg).__name__)
print("Token usage:", last_msg.usage_metadata)  # se existir
```

### Resumo

- Esse objeto é o histórico completo da conversa do agente.
- A maioria das vezes você não quer mostrar todo o histórico (tool calls, pensamentos intermediários, etc.).
- Por isso pegamos sempre o `[-1]` (última mensagem), que normalmente é a resposta final da IA para o usuário.
